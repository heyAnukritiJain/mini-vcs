# Mini Version Control System (Mini-VCS)

A lightweight snapshot-based version control system built using C++, MySQL, and basic web technologies.

---
# Overview

Mini-VCS is a ground-up implementation of core version control concepts — built without Git or any external VCS library. It allows developers to snapshot a project directory, log commits with metadata to a MySQL database, roll back to any prior state atomically, and visualize version history through both a CLI and a browser-based web interface.
Key technical decisions:

Used MySQL C API directly (no ORM) for fine-grained control over commit transactions and rollback atomicity
Designed snapshots as full directory copies with metadata stored in MySQL, separating file state from commit history
Exported version history to JSON to decouple the web viewer from the database, enabling static hosting
Chose a Python HTTP server for the web UI to keep the stack minimal and dependency-free

This project demonstrates systems-level thinking: designing data consistency guarantees, handling filesystem operations safely, and building a full-stack developer tool from scratch.

## 🚀 Features
* Create snapshots of a project directory
* Store commit messages and metadata in MySQL
* View version history via command line
* Export version history to JSON
* Display version history using a simple web interface

# Architecture

CLI (C++ binary)
      ↓
  VCS Core Engine
  ├── Snapshot Manager   → Copies project files to versioned storage
  ├── Commit Logger      → Writes metadata to MySQL (atomic transactions)
  ├── Rollback Engine    → Restores file state from snapshots
  └── JSON Exporter      → Writes version history to versions.json
                                    ↓
                          Web UI (HTML + JS)
                          Reads versions.json via local HTTP server

# CLI Commands

# Initialize VCS in current directory
./vcs init

# Take a snapshot with a commit message
./vcs commit "your commit message"

# View full commit history
./vcs log

# Roll back to a specific version
./vcs rollback <version_id>

# Export history to JSON
./vcs export

## ▶️ How to Run

1. Clone the repository
```
git clone https://github.com/heyAnukritiJain/mini-vcs.git
cd mini-vcs
```

2. Set up the database
```
mysql -u root -p
CREATE DATABASE mini_vcs;
USE mini_vcs;
-- Run schema from database/schema.sql
```

3. Compile the CLI
```
g++ -std=c++17 vcs_core/main.cpp -o vcs $CPPFLAGS $LDFLAGS -lmysqlclient
```

4. Run the CLI
```
./vcs init
./vcs commit "Initial commit"
./vcs log
```

5. Launch the web viewer
```
bashcd web
python3 -m http.server 8000
```
Open http://localhost:8000 in your browser.
